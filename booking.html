预约页面

通用检测

  • PC端根据PC端专题快速掌握
  • 移动端根据移动端专题快速掌握

预约人数

预约人数必须为活动字体，不能为图片

预约进度

如页面有预约人数进度模块，需要能够通过改变dom的CSS类名控制高亮效果

登录

点击登录弹出选择微信或手Q登录的弹窗，登录后的样式无错位

预约按钮

页面的预约按钮，需要有已经预约状态的样式

.btn-booking{}          //预约前
.btn-booking-disabled{} //预约后

复制链接

预约成功后，点击邀请链接按钮能成功一键复制

手机验证

填入手机号码点确定后，获取验证码按钮需要变成60秒倒计时，倒计时完成后才能重新点击

QQ聊天窗口分享

PC页面在添加以下meta，用于PC端QQ 聊天窗口分享（相关信息请联系接口人）

<meta itemprop="name" content="标题：必填 长度至少为4" />
<meta itemprop="description" content="摘要：必填 长度至少为10" />
<meta itemprop="image" content="图片地址：可为空" />

社区入口

移动端预约页面需要根据useragent用脚本控制，页面的社区入口显示和隐藏。 在微信展示时，隐藏兴趣部落 在手Q展示时，隐藏微信公众号 示例： 

    var ua = navigator.userAgent.toLowerCase();//获取判断用的对象
    if (ua.match(/MicroMessenger/i) == "micromessenger") { 
    //微信打开，隐藏兴趣部落
    } else if (/qq\/(\/[\d\.]+)*/.test(ua) || /qzone\//.test(ua)) {
    //手Q打开，隐藏公众号二维码
    } else {
        //其他浏览器打开全部展示
    }

抽奖信息

  1. 抽奖图片如需要经常更换、需要单独切割
  2. 引用公用抽奖组件组件，点击抽奖后能有动画演示效果，页面无报错信息
  3. 奖品图片需要和奖品的文字描述一致，文字不能有错别字

落地页

如果是落地页专题替换官网引导页，确认页面上《进入官网》的链接需要跳转到主页，不能直接跳转域名，避免死循环
